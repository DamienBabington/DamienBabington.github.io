The following panels show some of the skills used to create this app.

GitHub

Mapkit:

  • Draw polylines from starting location to destination
  • Calculate route distance and implement a pricing model based on the distance and ride type
  • Location search
  • Access passenger and driver locations with CoreLocation
  • Display driver locations on map using custom annotations (only when signed in as a passenger)
  • Use a MapViewRepresentable which can be rendered in SwiftUI, allowing access to required UIKit functionality

Side Menu & Settings:

  • Sliding side Menu View
  • Settings List View
  • This is where users can save locations as their Home or Workplace and sign out of the app

Firebase:

  • User authentication
  • Storing and fetching data with Firestore database
  • Observe changes to specific fields in the database (SnapshotListener)
  • Differentiate between driver accounts and passenger accounts

Architecture:

  • MVVM

SwiftUI & State Management:

  • Used enumerations to manage the state of the map and the state of a trip/ride
  • Dark Mode support
  • Refactored reusable views for cleaner, modularized code
  • Manage trip cancellation from either the driver or passenger
  • @Published and @Binding properties
  • .onReceive modifiers
